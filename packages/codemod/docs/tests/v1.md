## Bugs discovered

### Dependency upgrade crash on npm aliases

`cool-admin-midway` uses `"typeorm": "npm:@cool-midway/typeorm@0.3.20"` and `twenty` uses `"typeorm": "patch:typeorm@0.3.20#./patches/typeorm"`. Both crash the `semver.minVersion()` call with `TypeError: Invalid comparator`. The dependency upgrade should handle non-standard version specifiers gracefully.

### `datasource-mssql-domain` false positives

The `datasource-mssql-domain` transform matches any object property named `domain` and is not scoped to TypeORM config. It fires on cookie configs (`foal`), OAuth configs (`hoppscotch`), NestJS configs (`nest-admin`), and test fixtures (`nestjsx-crud`). This needs the same `fileImportsFrom("typeorm")` guard applied to `datasource-mongodb`.

### Parse errors

Most parse errors fall into three categories:

1. **Babel `constructor` keyword bug** — `constructor` as object property key (Vendure telemetry, minified JS files)
2. **TypeScript 5.x features** — `using` declarations, `satisfies` operator, decorator metadata (`hoppscotch`, `ever-gauzy`)
3. **Template files** — CLI generators with placeholder syntax (`foal` templates)

None of these affect transform correctness — the files either don't import from `typeorm` or use syntax the Babel parser can't handle.

## Analysis

### What works well

- `find-options-string-relations` is the most applied transform (271 files across all projects) — correctly converts string arrays to object syntax
- `connection-to-datasource` hits 320+ files with zero false positives after the scoping fix
- `global-functions` correctly identifies deprecated API usage in older v0.2.x projects
- Dependency upgrades work reliably for standard version specifiers
- No false positives on wrapper classes (e.g. Vendure's `TransactionalConnection`)

### What needs improvement

- **`datasource-mssql-domain`**: needs TypeORM import scoping (same fix as `datasource-mongodb`)
- **Dependency upgrade**: needs to handle npm aliases (`npm:`) and patch protocols (`patch:`) gracefully
- **`datasource-sap`**: fires on `hoppscotch` which doesn't use TypeORM — likely needs TypeORM import scoping too
