## Known limitations

### Parse errors

Most parse errors fall into three categories:

1. **TypeScript 5.x features** — `using` declarations, `satisfies` operator, decorator metadata (`hoppscotch` 104 errors, `ever-gauzy` 23 errors)
2. **Template files** — CLI generators with placeholder syntax (`foal` 11 errors)
3. **Babel `constructor` keyword bug** — `constructor` as object property key (minified JS files)

None affect transform correctness — the files either don't import from `typeorm` or use syntax the Babel parser can't handle.
